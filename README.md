# Snake Game - local LLM

---

## Description

This repository contains the different snake games in HTML/CSS/JS implemented by 18 large language models (LLMs). These LLMs were installed locally using Ollama CLI, and the games were generated by them on the local PC. Below is the table of all LLMs with the Ollama installation links and their sizes in gigabytes.

| Criteria | Description |

| **UI/UX Design** | Visual appeal, styling, user feedback, game over screens |
| **Features** | Extra features: pause, restart, high score, mobile controls, responsive design |
| **Bug Free** | Absence of bugs, proper game state management |

## Detailed Analysis

### 1. deepseek-r1 8b - Score: 8.6/10 ⭐
**Files:** `index.html` (single file, 525 lines)

**Strengths:**
- Most feature-complete game with excellent UI/UX
- DOM-based grid rendering with CSS styling
- Mobile touch controls (D-pad buttons)
- Pause/Resume functionality
- High score tracking
- Responsive design with media queries
- WASD + Arrow keys support
- Speed increases as snake grows
- Clean game over screen with restart option
- Well-structured code with comments

**Weaknesses:**
- DOM-based rendering is less performant than Canvas
- Food position calculation has minor edge case issues

### Key Observations
1. **Model size matters**: Larger models (8b+) generally produced more complete and functional games
2. **Single-file vs multi-file**: Both approaches can work well; deepseek-r1 8b excelled as a single file
3. **Common missing features**: Most games lacked high score persistence and mobile controls
4. **Smaller models struggle**: Models under 2b produced non-functional or severely broken code
5. **Canvas vs DOM**: Canvas-based games were generally more performant and cleaner to implement
